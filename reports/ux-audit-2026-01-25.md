## React Performance Patterns

Based on analysis of Vercel React best practices.

### Data Fetching

**Finding #5 (HIGH PRIORITY):** Scout detail page has sequential database queries (waterfall).

**Location:** `src/app/(dashboard)/scouts/[id]/page.tsx`

**Current Flow (Sequential):**
```
1. getUser() ─────────────────────────────────>
2. Scout query ───────────────────────────────>
3. Profile query ─────────────────────────────>
4. Membership query ──────────────────────────>
5. Transactions query ────────────────────────>
6. Guardians query ───────────────────────────>
7. Available profiles query ──────────────────>
8. Advancement progress ──────────────────────>
```
Total: ~5-6 sequential round trips

**Recommended Flow (Parallel where possible):**
```
1. getUser() ─────────────────────────────────>
2. [Scout + Profile + Membership] ─────────── Promise.all()
3. [Transactions + Guardians + Advancement] ─ Promise.all()
```

**Positive Examples (Already Using Promise.all):**
- `advancement/page.tsx:59-135` - 5 parallel queries
- `payments/page.tsx:108-136` - 3 parallel queries
- `billing/page.tsx:96-142` - 2 parallel queries

**Severity:** High
**Effort:** Medium (restructure queries, maintain auth guards)
